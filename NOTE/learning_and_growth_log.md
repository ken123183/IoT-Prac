## 1. 核心技術成長 (Hard Skills)

### 🚀 分散式系統與併發控制
*   **學習點**：以往可能只處理簡單的 CRUD，但在這個專案中，我透過 **PostgreSQL 悲觀鎖 (`SELECT FOR UPDATE`)** 解決了真實世界中的資源競爭問題。
*   **心得**：我理解了「強一致性」在資產交易中的不可替代性，以及如何在 Spring Boot 中優雅地實施事務保護。

### ☁️ 雲端原生運維 (Cloud Native)
*   **學習點**：我掌握了 **GCP Cloud Run (Serverless)** 的部署與維優，尤其是如何解決雲端網路隔離 (VPC Connector) 的通訊問題。
*   **心得**：我學會了如何觀測雲端日誌與指標來診斷性能瓶頸（例如：503 錯誤與 Socket 400 報錯），並進行相應的實例規格與併發參數調過。

### 📡 異步通訊與性能工程
*   **學習點**：深入理解了 **RabbitMQ** 的 Ack 機制與 **Redis** 的連線池優化。
*   **心得**：我知道了如何透過「削峰填谷」來保護脆弱的後端資料庫，這讓我的架構設計思維從「能跑就好」進化到了「高可用與高效能」。

## 2. 軟實力與解決問題能力 (Soft Skills)

### 🧩 複雜故障排除 (Troubleshooting)
*   **案例**：在部署到 GCP 時，Socket.io 因為 Load Balancer 的 Polling 問題導致連線失效。
*   **成長**：我沒有盲目搜索錯誤訊息，而是分析了 **WebSocket 握手協定** 與 **Serverless 負載平衡機制** 的底層衝突，最終以「強制 WebSocket 傳輸」這一精準的手段解決問題。這展現了我具備從底層原理推導問題核心的能力。

### 🏗️ 從零到一的架構規劃
*   **成長**：我不僅完成了代碼編寫，還撰寫了完整的 **README**、**GCP Runbook** 與 **技術演進藍圖**。這展現了我具備完備的文檔意識，這在團隊協作中是非常寶貴的資產。

## 3. 自我反思：如果能重來一次？

作為自我挑戰，如果我有更多的時間或預算，我會：
1.  **導入 Terraform (IaC)**：將目前手動執行的 `gcloud` 命令自動化。
2.  **增加全方位監測**：接入 **Prometheus/Grafana** 儀表板，即時監看各微服務的健康狀況。
3.  **單元測試覆蓋率**：為 `core-service` 的租借邏輯增加更多的 JUnit 測試與整合測試。
